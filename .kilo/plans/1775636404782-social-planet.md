## Objetivo

Implementar funcionalidades para enlaces de redes sociales y embeds (YouTube, Twitter, etc.)

## Estado Actual

- ✅ v0.1.0 - Foundation (auth, Next.js setup, Prisma)
- ✅ v0.2.0 - Core Links (CRUD completo de links)

## Siguiente: v0.5.0 - SOCIAL FEATURES

### Backend

| ID  | Endpoint                   | Método | Descripción             |
| --- | -------------------------- | ------ | ----------------------- |
| S1  | GET /api/links/types       | GET    | Lista de tipos de links |
| S2  | POST /api/links/embed      | POST   | Guardar embed metadata  |
| S3  | GET /api/links/:id/preview | GET    | Obtener preview de URL  |

### Tipos de Links Soportados

- YouTube (embed)
- Twitter/X (embed)
- Instagram (embed)
- TikTok (embed)
- Spotify (embed)
- LinkedIn
- GitHub
- Custom (URL normal)

### Frontend

| ID  | Componente       | Descripción              |
| --- | ---------------- | ------------------------ |
| C1  | LinkTypeSelector | Selector de tipo de link |
| C2  | LinkEmbedPreview | Preview del embed        |
| C3  | SocialButton     | Botón de red social      |
| C4  | EmbedRenderer    | Renderizador de embeds   |

### Flujo de Implementación

#### Fase 1: Tipos de Links

1. Crear endpoint `/api/links/types` con tipos disponibles
2. Actualizar modelo Link para incluir `linkType` y `embedData`
3. Crear LinkTypeSelector component

#### Fase 2: Embeds

1. Crear endpoint para preview de URLs
2. Implementar detección de tipo de URL (YouTube, Twitter, etc.)
3. Crear componente EmbedRenderer
4. Actualizar LinkForm para soporta embeds

#### Fase 3: Social Buttons

1. Crear SocialButton component
2. Agregar estilos específicos por red social
3. Actualizar public profile para renderizar embeds

### Definition of Done

- [ ] Links de YouTube se muestran como embed
- [ ] Links de Twitter/X se muestran como embed
- [ ] Preview de enlaces sociales
- [ ] Selector de tipo de link en el formulario
